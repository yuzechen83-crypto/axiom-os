# Axiom-OS 本地部署（在您电脑上运行）

## 1. 环境要求

- **Python 3.10+**（推荐 3.10 或 3.11）
- Windows / macOS / Linux 均可

## 2. 一键安装与验证

在项目根目录打开终端（PowerShell 或 CMD），执行：

```powershell
# 进入项目目录（若尚未进入）
cd c:\Users\ASUS\PycharmProjects\PythonProject1

# 安装依赖
pip install -r requirements.txt

# 全部组件已合并至 axiom_os，无需额外安装。

# 快速验证（约 30 秒）：基准测试 + 报告
python -m axiom_os.benchmarks.run_benchmarks --config quick --report
```

验证通过后，报告在：`axiom_os\benchmarks\results\`（含 `benchmark_report.md`、`benchmark_report.html`）。

## 3. 常用命令

| 用途           | 命令 |
|----------------|------|
| 全流程验证     | `python -m axiom_os.validate_all` |
| 公式发现 Demo  | `python -m axiom_os.demos.discovery_demo` |
| 快速基准       | `python -m axiom_os.benchmarks.run_benchmarks --config quick --report` |
| 标准基准       | `python -m axiom_os.benchmarks.run_benchmarks --config standard --report` |
| Chat 界面（LLM 意图层） | `streamlit run axiom_os/agent/chat_ui.py` |
| Dashboard      | `streamlit run axiom_os/dashboard.py` |
| 仅 RAR         | `python -m axiom_os.validate_all --rar` |
| 仅电池 RUL     | `python -m axiom_os.validate_all --battery` |
| 湍流 PINN-LSTM | `python -m axiom_os.validate_all --turbulence-pinn-lstm` |
| JHTDB LES-SGS | `python -m axiom_os.experiments.jhtdb_les_sgs` 或 `--jhtdb-les-sgs` |
| 公式结晶 | `python -m axiom_os.experiments.crystallize_formulas --rar-only` 或 `--crystallize` |
| Lorenz PINN-LSTM | `python -m axiom_os.experiments.run_lorenz_pinn_lstm` 或 `--lorenz-pinn-lstm` |
| Feynman 基准 | `python -m axiom_os.experiments.benchmark_feynman` |

| 暗物质发现 | `python -m axiom_os.validate_all --dark-matter` 或 `python -m axiom_os.experiments.dark_matter_discovery` |

## 4. 可选：虚拟环境（推荐）

避免与系统 Python 混用，可使用 venv：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1   # PowerShell
# 或: .venv\Scripts\activate.bat   # CMD
pip install -r requirements.txt
python -m axiom_os.benchmarks.run_benchmarks --config quick --report
```

## 5. Chat 与 LLM 层

Chat 界面支持两种引擎：

- **本地意图**：用自然语言说「跑一下 RAR」「跑基准」「公式发现」「全流程验证」「电池 RUL」等，会先识别意图再调度对应 Axiom 模块。未配置 `OPENAI_API_KEY` 时使用关键词回退。
- **DeepSeek 扩展**：侧边栏勾选「使用 DeepSeek 扩展」后，接入 DeepSeek 大模型（axiom-os 配置），可调用工具收集不同领域数据、运行基准/RAR/Discovery、并扩展或优化 Axiom。需设置环境变量：
  - `DEEPSEEK_API_KEY` 或 `AXIOM_OS_LLM_KEY`（从 [DeepSeek 开放平台](https://platform.deepseek.com/) 获取）
  - 示例（PowerShell）：`$env:DEEPSEEK_API_KEY="sk-your-key"`

## 6. 故障排除

- **ModuleNotFoundError: sklearn**：执行 `pip install scikit-learn`
- **RCLN 报错**：检查依赖与版本，全部组件已开源可调取
- **CUDA/GPU**：未安装 CUDA 时自动用 CPU，不影响基准与 Discovery
